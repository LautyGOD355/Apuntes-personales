SQL: Introducción y Fundamentos

¿Qué es SQL?

SQL (Structured Query Language) es un lenguaje de programación utilizado para administrar y manipular bases de datos relacionales. Permite interactuar con la información almacenada en una base de datos, realizar consultas, agregar, actualizar o eliminar datos, y mucho más.

¿Por qué SQL es Importante?

SQL desempeña un papel crucial en la gestión de datos en aplicaciones web y sistemas empresariales. Es ampliamente utilizado en la industria para:

  • Recuperar información específica de una base de datos.
  • Actualizar registros existentes.
  • Insertar nuevos datos en una base de datos.
  • Eliminar información no deseada.
  • Crear y modificar estructuras de base de datos, como tablas e índices.

Características Clave de SQL

SQL se destaca por sus características clave:

  • Consistencia y Estructura: Las bases de datos relacionales gestionadas con SQL mantienen una estructura coherente y organizada.
  • Lenguaje Estándar: SQL es un lenguaje estándar reconocido internacionalmente, lo que facilita la comunicación entre diferentes sistemas de bases de datos.
  • Flexibilidad: Permite la realización de consultas complejas para extraer datos específicos.

SELECT: Obtener Datos de una Tabla

El Comando SELECT

El comando `SELECT` es esencial en SQL y se utiliza para obtener datos de una tabla. Puede recuperar información específica de una o varias tablas en una base de datos. Aquí hay algunos aspectos clave sobre `SELECT`:

Selección de Columnas

Con `SELECT`, puedes elegir qué columnas deseas recuperar de una tabla. Puedes seleccionar columnas específicas o incluso todas las columnas de la tabla.

Ejemplo de Uso

Un ejemplo sencillo de una consulta `SELECT` sería:


	    SELECT nombre, apellido
	    FROM empleados;

Esta consulta recuperaría los valores de las columnas "nombre" y "apellido" de la tabla "empleados".

Uso de Alias

Además, puedes usar alias para renombrar las columnas seleccionadas en los resultados de la consulta. Por ejemplo:


	    SELECT nombre AS NombreEmpleado, apellido AS ApellidoEmpleado
	    FROM empleados;

Esto cambiaría los encabezados de las columnas en los resultados a "NombreEmpleado" y "ApellidoEmpleado".

Structured Data: Datos Estructurados

¿Qué son los Datos Estructurados?

Los datos estructurados, también conocidos como 'structured data', son un tipo de información que se organiza de manera tabular y se almacena en filas y columnas dentro de una base de datos relacional.

Ejemplo

Imagina una tabla llamada "empleados" con las siguientes columnas: "nombre", "apellido", "edad" y "cargo". Todos estos datos se consideran datos estructurados porque están organizados de manera ordenada en la tabla y cada columna tiene un nombre y un tipo de datos específico.

¿Qué son los Datos No Estructurados?

Los datos no estructurados, también conocidos como 'unstructured data', son información que no se ajusta a un formato tabular y no se puede organizar fácilmente en filas y columnas dentro de una base de datos relacional.

Ejemplo

Los datos no estructurados incluyen archivos de audio, video, imágenes, documentos complejos y otros tipos de información que no pueden ser representados de manera ordenada en una tabla de base de datos.

Consultas de SQL: Concepto Clave

¿Qué son las Consultas en SQL?

Las consultas en SQL son instrucciones escritas en lenguaje SQL que se envían a una base de datos para recuperar, actualizar o manipular datos. Estas consultas permiten realizar operaciones específicas en la base de datos, como buscar información, filtrar resultados y combinar datos de diferentes tablas.

Ejemplo

Un ejemplo de consulta SQL podría ser: "SELECT nombre, apellido FROM clientes WHERE edad > 25". Esta consulta seleccionaría los nombres y apellidos de clientes cuya edad sea mayor que 25 años. Las consultas son esenciales para interactuar con una base de datos de manera efectiva.

Bases de Datos Relacionales

¿Qué son las Bases de Datos Relacionales?

Las bases de datos relacionales son sistemas de gestión de bases de datos que utilizan tablas para organizar la información. Estas tablas pueden tener relaciones entre sí mediante el uso de claves primarias y claves foráneas, lo que facilita el acceso y la gestión de datos.

Ejemplo

Un ejemplo de una base de datos relacional podría ser una base de datos de una tienda en línea, donde hay tablas para productos, clientes, pedidos y más. Estas tablas se relacionan para mantener la información organizada y permitir consultas complejas.

Campos Clave en Bases de Datos Relacionales

¿Qué son los Campos Clave?

Los campos clave son columnas especiales en una tabla que contienen valores únicos para cada fila. Estos valores permiten identificar de manera única cada registro en la tabla y establecer relaciones entre diferentes tablas en una base de datos relacional.

Ejemplo

En una tabla de clientes, el número de identificación del cliente podría ser un campo clave, lo que garantiza que cada cliente tenga un valor único en esa columna. Esto facilita la búsqueda y la vinculación de información en toda la base de datos.

Uso de Asterisco (*) en SQL

¿Qué Significa el Asterisco (*)?

El símbolo asterisco (*) se utiliza en SQL como un comodín para seleccionar todas las columnas de una tabla en una consulta sin tener que enumerar cada columna individualmente. Esto ahorra tiempo y simplifica las consultas.

Ejemplo

En una consulta SQL, "SELECT * FROM productos" seleccionaría todas las columnas de la tabla de productos. Es útil cuando se necesita recuperar toda la información de una tabla sin especificar cada columna por separado.

Comentarios en Código SQL

¿Por qué usar Comentarios en Código SQL?

Los comentarios en el código SQL son útiles para agregar explicaciones o notas que ayuden a comprender el propósito de una consulta o instrucción. Estos comentarios no afectan la ejecución de las consultas y son especialmente valiosos en código complejo o colaborativo.

Ejemplo


	-- Este es un comentario de una sola línea en SQL
	SELECT nombre, apellido
	FROM clientes
	WHERE edad > 25;

Sensibilidad de Mayúsculas y Minúsculas en SQL

¿Qué Significa la Sensibilidad de Mayúsculas y Minúsculas en SQL?

SQL es un lenguaje que no distingue entre mayúsculas y minúsculas (case-insensitive), lo que significa que las palabras clave y los nombres de objetos en SQL pueden escribirse en mayúsculas o minúsculas indistintamente. Esto simplifica la escritura de consultas, ya que no es necesario preocuparse por la capitalización.

Ejemplo

En SQL, "SELECT" y "select" se tratan de la misma manera y se pueden usar de forma intercambiable. Esto significa que las siguientes dos consultas son equivalentes:


	SELECT nombre FROM clientes;
	select nombre from clientes;

Ordenar Resultados con "ORDER BY"

¿Qué es "ORDER BY" en SQL?

"ORDER BY" es una cláusula en SQL que se utiliza para organizar los resultados de una consulta en un orden específico. Puede ordenar los resultados en orden ascendente (de menor a mayor) o descendente (de mayor a menor) basándose en los valores de una o más columnas.

Ejemplo

Supongamos que tenemos una tabla de productos y deseamos mostrarlos en orden alfabético por su nombre. La consulta SQL sería:


	SELECT nombre, precio
	FROM productos
	ORDER BY nombre ASC;

Esto mostraría la lista de productos ordenados alfabéticamente por sus nombres en orden ascendente.

Operadores Lógicos en SQL

¿Qué son los Operadores Lógicos en SQL?

En SQL, los operadores lógicos permiten combinar condiciones para filtrar y recuperar datos de manera más precisa en las consultas. Los operadores lógicos más comunes son "AND", "OR" y "NOT".

Ejemplo

Supongamos que queremos seleccionar todos los clientes que sean mayores de 25 años y vivan en Nueva York. Podríamos usar el operador "AND" en la cláusula "WHERE" de la siguiente manera:


	SELECT nombre, edad, ciudad
	FROM clientes
	WHERE edad > 25 AND ciudad = 'Nueva York';

Esta consulta seleccionaría clientes que cumplan ambas condiciones: tener más de 25 años y vivir en Nueva York.

La Cláusula "WHERE" en SQL

¿Qué es la Cláusula "WHERE" en SQL?

La cláusula "WHERE" se utiliza en SQL para filtrar registros en una consulta. Permite especificar condiciones que deben cumplirse para que un registro sea incluido en los resultados. Es fundamental para recuperar datos específicos de una tabla.

Ejemplos de Uso

Filtrar por Valor Mayor que

Imagina que deseamos seleccionar todos los productos cuyo precio sea superior a $50. La consulta SQL sería:


	SELECT nombre, precio
	FROM productos
	WHERE precio > 50;

Esta consulta devolvería los productos que tienen un precio superior a $50.

Filtrar por Valor Igual a

Si queremos obtener información sobre un producto específico, como uno con un ID de producto igual a 101, la consulta sería:


	SELECT nombre, precio
	FROM productos
	WHERE id_producto = 101;

Esta consulta seleccionaría el producto con ID 101.

Filtrar por Texto

Para buscar productos cuyo nombre contenga la palabra "camiseta", puedes usar el operador "LIKE" de la siguiente manera:


	SELECT nombre
	FROM productos
	WHERE nombre LIKE '%camiseta%';

Esto seleccionaría productos con nombres que contienen la palabra "camiseta" en cualquier posición.

Filtrar por Valor NULL

Si deseas encontrar registros donde un campo esté vacío, puedes usar "IS NULL". Por ejemplo, para encontrar empleados sin un número de teléfono registrado:


	SELECT nombre
	FROM empleados
	WHERE telefono IS NULL;

Esto seleccionará empleados sin número de teléfono.

Filtrar por Valor NO NULL

Para encontrar registros con un valor en un campo, puedes usar "IS NOT NULL". Por ejemplo, para encontrar empleados con dirección de correo electrónico:


	SELECT nombre
	FROM empleados
	WHERE correo_electronico IS NOT NULL;

Esto seleccionará empleados con dirección de correo electrónico registrada.

Filtrar por Valor Menor que o Igual a


	SELECT nombre, precio
	FROM productos
	WHERE precio <= 30;

Filtrar por Rango de Fechas


	SELECT nombre, fecha_compra
	FROM pedidos
	WHERE fecha_compra BETWEEN '2023-01-01' AND '2023-12-31';

Filtrar por Varias Condiciones


	SELECT nombre, categoria
	FROM productos
	WHERE precio > 100 AND categoria = 'Electrónica';

Filtrar por Valores en una Lista


	SELECT nombre, categoria
	FROM productos
	WHERE categoria IN ('Electrónica', 'Ropa', 'Juguetes');

Filtrar por Valores que no Están en una Lista


	SELECT nombre, categoria
	FROM productos
	WHERE categoria NOT IN ('Electrónica', 'Ropa', 'Juguetes');

Filtrar Usando Wildcards


	SELECT nombre
	FROM empleados
	WHERE nombre LIKE 'M%';

Filtrar por Fechas Anteriores


	SELECT nombre, fecha_nacimiento
	FROM personas
	WHERE fecha_nacimiento < '2000-01-01';

Filtrar por Texto y Otra Condición


	SELECT nombre, categoria
	FROM productos
	WHERE nombre LIKE '%oferta%' AND precio < 50;

El Operador "LIKE" en SQL

¿Qué es el Operador "LIKE" en SQL?

El operador "LIKE" se utiliza en SQL para buscar patrones en los datos de una columna. Es especialmente útil cuando deseas buscar valores que contienen una cadena específica o patrón en una columna.

Ejemplo

Supongamos que deseamos encontrar todos los nombres de productos que contienen la palabra "cámara". Podemos usar "LIKE" de la siguiente manera:


	SELECT nombre
	FROM productos
	WHERE nombre LIKE '%cámara%';

Esta consulta seleccionará todos los productos cuyos nombres contengan la palabra "cámara".

Sensibilidad de Mayúsculas y Minúsculas en Patrones

¿Qué Debes Saber sobre la Sensibilidad de Mayúsculas y Minúsculas en Patrones?

Algunos patrones en SQL pueden ser sensibles a las mayúsculas y minúsculas, lo que significa que distinguen entre letras mayúsculas y minúsculas. Sin embargo, puedes realizar búsquedas sin tener en cuenta la capitalización utilizando las funciones "LOWER()" y "UPPER()".

Ejemplo

Para buscar títulos en una tabla de libros que contengan la palabra "cooking" sin importar la capitalización, puedes hacer lo siguiente:


	SELECT title
	FROM books
	WHERE LOWER(title) LIKE '%cooking%';

Esta consulta ignorará la capitalización al buscar títulos que contengan "cooking".

Uso de Condiciones para Crear Nuevos Campos en SQL

¿Cómo se Usan Condiciones para Crear Nuevos Campos en SQL?

En SQL, puedes utilizar condiciones para verificar valores en una columna y crear nuevos campos basados en los resultados. Esto se hace usando la sintaxis "condición AS nombre_campo_nuevo".

Ejemplo

Supongamos que deseamos crear un nuevo campo llamado "result" que contenga valores booleanos (T o F) basados en si el año de una película es mayor que 2000. La consulta sería:


	SELECT year, year > 2000 AS result
	FROM movies;

Esto creará un nuevo campo "result" con valores "T" o "F" dependiendo de si el año es mayor que 2000.

AVG, SUM, MAX. Funciones de Agregación en SQL

¿Qué son las Funciones de Agregación en SQL?

Las funciones de agregación en SQL son utilizadas para realizar cálculos en conjuntos de datos. Estas funciones operan en múltiples filas para proporcionar un solo resultado. Algunas funciones de agregación comunes incluyen "SUM" (suma), "AVG" (promedio), "COUNT" (conteo) y "MAX" (máximo).

Ejemplo

Supongamos que queremos calcular la suma de los valores en la columna "cantidad_vendida" de una tabla de ventas. La consulta SQL sería:


	SELECT SUM(cantidad_vendida) AS total_vendido
	FROM ventas;
	    

	-- Suma de los precios de productos vendidos
	SELECT SUM(precio) AS total_ventas
	FROM ventas;
		

	-- Cálculo del promedio de edades de empleados
	SELECT AVG(edad) AS promedio_edad
	FROM empleados;
		

	-- Valor máximo y mínimo de precios de productos
	SELECT MAX(precio) AS precio_maximo, MIN(precio) AS precio_minimo
	FROM productos;
		

	-- Conteo total de registros de usuarios
	SELECT COUNT(*) AS total_registros
	FROM usuarios;

Esto nos dará el total de la cantidad vendida en todas las filas de la tabla de ventas.

Agrupación de Datos con GROUP BY

¿Qué es la Agrupación de Datos con GROUP BY en SQL?

La cláusula "GROUP BY" en SQL se utiliza para agrupar filas que tienen los mismos valores en una o más columnas. Generalmente se combina con funciones de agregación como SUM, COUNT, AVG, MAX o MIN para realizar cálculos sobre los grupos de datos.

Ejemplo

Supongamos que tenemos una tabla de empleados y queremos saber cuántos empleados hay en cada departamento:


	SELECT departamento, COUNT(*) as cantidad_empleados
	FROM empleados
	GROUP BY departamento;

Esta consulta devolvería el número de empleados en cada departamento.

La Cláusula HAVING en SQL

¿Qué es la Cláusula HAVING en SQL?

La cláusula "HAVING" en SQL se utiliza junto con la cláusula "GROUP BY" para filtrar grupos de datos después de haberse aplicado una función de agregación. Permite establecer condiciones en los resultados de las funciones de agregación.

Ejemplo

Supongamos que deseamos encontrar los departamentos con un promedio de salario superior a $60,000:


	SELECT departamento, AVG(salario) as salario_promedio
	FROM empleados
	GROUP BY departamento
	HAVING AVG(salario) > 60000;

Esta consulta devolvería los departamentos que cumplen con la condición de tener un salario promedio superior a $60,000.

El Comando DISTINCT en SQL

¿Qué es el Comando DISTINCT en SQL?

El comando "DISTINCT" en SQL se utiliza para eliminar valores duplicados en una consulta y garantizar que solo se muestren resultados únicos. Se coloca al lado del operador "SELECT" y afecta a una o más columnas en la salida.

Ejemplo

Supongamos que tenemos una tabla de clientes y deseamos obtener una lista de ciudades únicas:


	SELECT DISTINCT ciudad
	FROM clientes;

Esta consulta mostrará una lista de ciudades únicas, eliminando cualquier repetición.

NULL en SQL: Ausencia de Valor

¿Qué es NULL en SQL?

En SQL, "NULL" se utiliza para indicar la ausencia de un valor en una columna de datos. Representa la condición en la que la información no está disponible o no se ha ingresado en la base de datos para un campo específico.

Ejemplo

Supongamos que tenemos una tabla de productos y algunos productos no tienen un valor asignado para su descripción:


	SELECT nombre, descripcion
	FROM productos
	WHERE descripcion IS NULL;

Esta consulta devolverá los productos que tienen un valor NULL en la columna de descripción.

Booleanos en SQL

¿Cómo se utilizan los Booleanos en SQL?

En SQL, los booleanos son valores lógicos que representan "verdadero" o "falso". Se utilizan en las cláusulas "WHERE" para realizar filtrados basados en condiciones lógicas.

Ejemplo

Supongamos que deseamos encontrar todos los productos que están en stock (cantidad mayor que cero):


	SELECT nombre, cantidad
	FROM productos
	WHERE cantidad > 0;

Esta consulta devolverá los productos que tienen una cantidad mayor que cero en stock, lo que representa "verdadero".

Operadores Lógicos en SQL: AND, OR, y NOT

¿Qué son los Operadores Lógicos en SQL?

En SQL, los operadores lógicos son herramientas esenciales que permiten combinar y evaluar condiciones en las consultas. Los operadores lógicos más comunes son "AND", "OR" y "NOT", y desempeñan un papel crucial en la filtración precisa de datos.

Operador "AND"

El operador "AND" se utiliza para combinar múltiples condiciones en una consulta. Su funcionamiento es simple: devolverá filas que cumplan con todas las condiciones especificadas. En otras palabras, todas las condiciones unidas por "AND" deben ser verdaderas para que una fila se incluya en los resultados.

Ejemplo de Operador "AND"

Supongamos que deseamos encontrar productos que estén en stock y tengan un precio inferior a $50:


	SELECT nombre, cantidad
	FROM productos
	WHERE cantidad > 0 AND precio < 50;

Esta consulta devolverá productos que cumplen ambas condiciones: tener una cantidad mayor que cero y un precio inferior a $50.

Operador "OR"

El operador "OR" se utiliza para combinar múltiples condiciones en una consulta. A diferencia de "AND", "OR" devolverá filas que cumplan al menos una de las condiciones especificadas. En otras palabras, si cualquiera de las condiciones es verdadera, la fila se incluirá en los resultados.

Ejemplo de Operador "OR"

Supongamos que deseamos encontrar productos que estén en stock o tengan un precio inferior a $50:


	SELECT nombre, cantidad
	FROM productos
	WHERE cantidad > 0 OR precio < 50;

Esta consulta devolverá productos que cumplen al menos una de las condiciones: tener una cantidad mayor que cero o un precio inferior a $50.

Operador "NOT"

El operador "NOT" se utiliza para negar una condición en una consulta. Cambia el resultado de una condición verdadera a falsa, y viceversa. Es útil cuando deseas encontrar registros que no cumplan con una cierta condición.

Ejemplo de Operador "NOT"

Supongamos que deseamos encontrar productos que no estén agotados (cantidad no igual a cero):


	SELECT nombre, cantidad
	FROM productos
	WHERE NOT cantidad = 0;

Esta consulta devolverá productos cuya cantidad no sea igual a cero, es decir, que no estén agotados.
